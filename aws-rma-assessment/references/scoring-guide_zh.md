# 评分指南 — 公式、级别和领域评分

## 总体成熟度评分

**公式**：（所有问题得分之和 / 总问题数 / 3）× 100

**评级标准**：
- 90-100%：优秀
- 75-89%：良好
- 60-74%：一般
- 45-59%：需改进
- <45%：关键风险

## 领域成熟度评分

- 计算10个主题领域的平均分
- 识别得分最低的3个领域作为优先改进领域

## P0关键风险专项摘要（必须出现在执行摘要中）

- 单独计算P0问题平均分：（P0得分之和 / P0题数 / 3）× 100
- 如果任何P0问题得分为Level 1，在执行摘要中添加 **"关键风险警告"** 醒目标识
- 在专项表格中列出所有Level 1的P0问题，包含领域、问题、当前级别和建议行动
- 确保关键风险不会被低优先级领域的高分掩盖

## 关键风险识别

- 所有P0问题得分为Level 1 → 高风险
- 所有P1问题得分为Level 1 → 中风险
- 按业务影响排序

## 优势领域识别

- 所有问题得分为Level 3
- 可作为组织最佳实践分享

---

## 成熟度级别定义

### Level 1 - 初始/临时
- 流程不规范或不存在
- 主要依赖人工操作
- 缺乏文档和自动化
- 高风险、低可预测性

### Level 2 - 可重复/已定义
- 有基本流程和文档
- 部分自动化
- 定期执行但可能不一致
- 中等风险、部分可预测

### Level 3 - 优化/管理
- 完全文档化和自动化的流程
- 定期通过演练和评审进行测试和验证
- 持续改进机制，有可衡量的成果
- 低风险、高可预测性
- 符合AWS最佳实践
- 注意：Level 3 的评估依据是**流程成熟度**（文档化、自动化、测试验证、持续改进），而非特定的数值阈值（如具体的 RTO 数值）。数值目标因业务场景而异。

## 特定领域评分指南

| 领域 | Level 1 | Level 2 | Level 3 |
|------|---------|---------|---------|
| **恢复目标** | 未定义或未正式记录 | 已定义但未定期验证 | 已定义、测试并持续监控 |
| **灾难恢复** | 无DR计划或未测试 | 有DR计划，定期测试 | 自动化DR，故障转移已验证，定期测试（季度+） |
| **监控与可观察性** | 基础监控，无统一日志 | 集中化日志，基本指标和告警 | 完整可观察性（日志、指标、追踪），主动告警 |
| **高可用性** | 单AZ，无冗余 | 多AZ部署，基本健康检查 | 多AZ自动故障转移，故障隔离边界已验证 |
| **变更管理** | 手动部署，无回滚策略 | 部分自动化，有基本回滚 | 全自动CI/CD，蓝绿/金丝雀部署 |
| **事件管理** | 无事件流程，临时响应 | 文档化运维手册，基本升级 | 自动化事件检测，结构化响应，无责任事后复盘 |
| **运营评审** | 无定期评审 | 定期评审（季度），基本指标 | 定期评审含行动跟踪，数据驱动决策 |
| **混沌工程与游戏日** | 无故障注入或演练 | 非生产环境偶尔演练 | 生产环境定期混沌实验，自动化稳态验证 |
| **组织学习** | 无韧性文化或培训 | 基本培训，非正式知识共享 | 韧性实践社区，持续培训，知识库 |
| **韧性分析** | 无依赖文档或故障建模 | 基本依赖映射，部分故障场景 | 全面依赖文档，故障场景建模，容量规划 |

## 跨技能评分对齐

与 `aws-resilience-modeling` 配合使用时：

| RMA 级别 | 近似 Modeling 星级 | 解释 |
|----------|-------------------|------|
| Level 1（初始） | 1-2 星 | 未实现或临时性 |
| Level 2（已定义） | 2.5-3.5 星 | 标准化，部分自动化 |
| Level 3（优化） | 4-5 星 | 优化，持续改进 |

> 注意：这是近似映射而非精确等价。RMA 评估组织成熟度（人员、流程、工具）；Modeling 评估单组件的技术实现深度。

### 与 AWS Resilience Hub 的关系

| 方面 | RMA 评估（本技能） | AWS Resilience Hub |
|------|-------------------|-------------------|
| **关注点** | 组织/流程成熟度（人员、流程、工具） | 技术配置合规性 |
| **范围** | 10 个领域 × 52 个问题，覆盖文化、治理、测试 | 每应用 RTO/RPO 策略、资源配置 |
| **输出** | 成熟度级别 (1-5) + 改进路线图 | 合规状态 + 建议操作 |
| **适用场景** | 首次韧性战略、成熟度基准评估 | 持续自动化合规监控 |

**推荐方法**：先通过 RMA 评估了解组织差距，然后使用 Resilience Hub 对你实施的技术改进进行持续自动化监控。
